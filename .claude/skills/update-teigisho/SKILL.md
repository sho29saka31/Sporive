# 要件定義書の更新

要件定義書（`docs/requirements.md`）に更新事項を反映するスキル。

## 手順

1. `docs/requirements.md` を読み、提示された更新事項が該当する章を特定する
2. 該当章の記述を更新する
   - 既存のフォーマット（表・見出し・箇条書きのスタイル）を維持する
   - 複数の章に影響する場合はすべての該当箇所を漏れなく更新し、記述間の矛盾を残さない
3. **必ず第14章「更新履歴」の表に行を追記する**（このスキルが使われたら省略不可）
   - 形式: `| 日付 | 変更内容 | 理由 |`
   - 日付は今日の日付（YYYY-MM-DD）
   - 「変更内容」は何をどう変えたかを簡潔に、「理由」はユーザーの提示内容から要約する。理由が提示されていない場合はユーザーに確認する
   - 追記は表の末尾（時系列順）
4. 更新内容が `docs/development-plan.md` や `CLAUDE.md` の記述と矛盾しないか確認し、影響がある場合はユーザーに報告する（勝手に書き換えず、更新するかを確認する）
5. 変更をコミットする（コミットメッセージに更新概要を含める）

## 注意事項

- 既存の決定事項（第14章の履歴で確定した内容）と矛盾する変更依頼の場合は、上書きしてよいかユーザーに確認する
- 未決事項（第13章）が今回の更新で確定した場合は、第13章から該当項目を削除し、本文の該当章に反映する
