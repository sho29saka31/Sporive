# mdファイルの更新

リポジトリ内の既存mdファイルに更新事項を反映するスキル。

## 手順

1. 更新対象のmdファイルを特定する
   - ユーザーがファイル名を明示していない場合は、更新内容から対象を推定する（`Glob` で `**/*.md` を確認）
   - 対象が曖昧な場合はユーザーに確認する
2. **対象が `docs/requirements.md`（要件定義書）の場合は、このスキルではなく `update-teigisho` スキルの手順に従う**（第14章「更新履歴」への追記が必須のため）
3. 対象ファイルを読み、提示された更新事項を反映する
   - 既存のフォーマット・文体・構成（見出しレベル、表のスタイル、箇条書き）を維持する
   - 更新によって同一ファイル内・他のmdファイルとの記述に矛盾が生じないか確認する
4. 他のmdファイルにも影響する更新の場合は、影響範囲をユーザーに報告し、あわせて更新するか確認する
5. 変更をコミットする（コミットメッセージに更新概要を含める）

## 注意事項

- 存在しないmdファイルの新規作成はこのスキルの対象外（通常の依頼として対応する）
- 更新事項が提示されていない状態でこのスキルが呼ばれた場合は、何を更新するかをユーザーに確認する
